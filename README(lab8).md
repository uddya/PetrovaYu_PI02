Вариант 2. Жадный алгоритм для составления расписания
Задача: реализовать жадный алгоритм для распределения работ на машины с минимизацией времени завершения (makespan).
Требования:
- Входные данные: число машин (n) и список длительностей работ
- Выход: распределение работ на машины и максимальное время завершения
- Сравнить с оптимальным решением для малых входов
Входные данные:
Число машин: 3
Длительности работ: [5, 7, 3, 8, 2, 6, 4, 9, 1, 5]

Работа алгоритма:
	Сортируем список работ по их длительности в порядке убывания. Причина такого шага заключается в том, что большие работы желательно назначать раньше, 
  чтобы равномерно заполнить свободные слоты на машинах.
jobs_sorted = sorted(durations, reverse=True)
	Затем начинаем назначать работы машинам по одному циклу. Для каждой работы ищем машину с минимальной текущей нагрузкой и назначаем эту работу на эту машину. 
  Машина с минимальной общей продолжительностью выполнения получает очередную работу.
machine_id = machines.index(min(machines))  
machines[machine_id] += job_time
	После назначения очередной работы соответствующая машина увеличивает своё суммарное время выполнения.
	В итоге получаем распределение работ по машинам и рассчитываем makespan — это максимальное время выполнения среди всех машин.
3. Временная сложность: O(mn), где m— количество работ, n— количество машин.
4. Объяснение временной сложности: так как сортировка выполняется однократно, а распределение работ повторяется многократно.
5. (2) Коэффициент аппроксимации.  Дайте определение коэффициента аппроксимации α для задачи минимизации. Что он характеризует?
Коэффициент аппроксимации (α) — это характеристика качества приближённого (жадного или эвристического) алгоритма в сравнении с наилучшим возможным решением оптимальной задачи минимизации.
Коэффициент аппроксимации позволяет оценить качество приближённого решения по отношению к теоретическому оптимальному решению. 
Если доказано, что α-алгоритм даёт гарантированную оценку относительной погрешности, то можно уверенно применять его в ситуациях, 
когда точное решение сложно или невозможно найти за приемлемое время.
